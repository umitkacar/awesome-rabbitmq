# Contributing to Awesome RabbitMQ

First off, thank you for considering contributing to Awesome RabbitMQ! It's people like you that make this resource truly awesome.

1. **Fork the repository**
2. **Create a new branch** for your addition
   ```bash
   git checkout -b add-resource-name
   ```
3. **Add your resource** in the appropriate section
4. **Commit your changes**
   ```bash
   git commit -m "Add [resource name]"
   ```
5. **Push to your fork**
   ```bash
   git push origin add-resource-name
   ```
6. **Open a Pull Request**

### âœ… Quality Guidelines

When adding a resource, please ensure:

#### For All Resources
- [ ] The link is valid and working
- [ ] The resource is relevant to RabbitMQ
- [ ] The description is clear and concise
- [ ] There are no spelling or grammatical errors
- [ ] The resource is placed in the correct category

#### For Tools & Libraries
- [ ] The project is actively maintained (commits within the last year)
- [ ] The project has clear documentation
- [ ] The project has a reasonable number of stars/users (if applicable)
- [ ] Include GitHub stars badge if it's a GitHub project

#### For Tutorials & Articles
- [ ] The content is accurate and up-to-date
- [ ] The tutorial is complete and tested
- [ ] The content adds value beyond existing resources

#### For Books & Courses
- [ ] Include author name
- [ ] Include publication year (if applicable)
- [ ] Mention if it's free or paid
- [ ] Add brief description of what's covered

### ğŸš« What NOT to Submit

Please avoid submitting:

- Duplicate resources already listed
- Outdated or deprecated tools
- Resources behind paywalls without clear indication
- Low-quality or incomplete tutorials
- Self-promotion without significant value
- Resources not directly related to RabbitMQ
- Dead links or abandoned projects

### ğŸ“‚ Category Guidelines

#### Choosing the Right Category

- **Getting Started**: Installation guides, basic setup, first steps
- **AI & ML Integration**: Machine learning, AI models, data processing
- **Architecture & Design Patterns**: EDA, CQRS, Saga, microservices patterns
- **Client Libraries & SDKs**: Language-specific clients
- **Tools & Plugins**: Extensions, management tools, utilities
- **Monitoring & Observability**: Prometheus, Grafana, logging, metrics
- **Learning Resources**: Official docs, blogs, articles
- **Tutorials & Courses**: Step-by-step guides, video courses
- **Performance & Optimization**: Tuning, benchmarks, scaling
- **Security Best Practices**: Authentication, encryption, hardening
- **Comparisons**: RabbitMQ vs other message brokers
- **Awesome Projects**: Real-world implementations, frameworks

If unsure, open an issue to discuss!
